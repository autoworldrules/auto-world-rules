##WHAT EACH ARGUMENT LOOKS LIKE AND WHAT IT MEANS
All four arguments are Python strings. They contain ASP-style text (facts and rules).

#ARGUMENT 1: cand_fact (str)
This is a single grounded candidate fact string you are scoring. It is exactly one atom ending in a period.
Example cand_fact:
maternal_aunt_or_uncle_of(2,3).
Grounded fact as it contains no variables. It uses concrete entity constants (integers like 2 and 3). The predicate name is lowercase (maternal_aunt_or_uncle_of).
Here maternal_aunt_or_uncle_of is the predicate. The fact states that entities 2 and 3 are related by this predicate.
You can view story building as a directed graph construction process: entities are nodes, facts are directed edges, and predicates are edge types.
In this view, adding cand_fact means adding a directed edge of type maternal_aunt_or_uncle_of from node 2 to node 3 to the existing story graph (see ARGUMENT 4).

Other candidate facts could be similar grounded atoms such as:
father_of(1,3).
living_in(3,4).
school_mates_with(0,1).
colleague_of(2,0).
not_living_in(1,4).

#ARGUMENT 2: definite_rules_program (str)
This is a string containing many definite (Horn) rules written in ASP syntax, usually one rule per line. Rules are typically ungrounded and may include variables like X, Y, Z.

Example definite_rules_program (small excerpt):
maternal_uncle_of(X,Y) :- maternal_aunt_or_uncle_of(X,Y), is_male(X,X).
aunt_or_uncle_of(X,Y) :- maternal_aunt_or_uncle_of(X,Y).
nibling_of(Y,X) :- aunt_or_uncle_of(X,Y).
living_in_same_place(X,Y) :- living_in_same_place(X,Z), living_in_same_place(Z,Y), !=(Y,X).
maternal_grandmother_of(V,X) :- paternal_grandmother_of(U,X), grandmother_of(V,X), !=(U,V).

How to read rules:
The head is on the left of ":-" and is a single atom.
The body is on the right of ":-" and is a comma-separated conjunction of atoms (and sometimes built-ins).
Intuition: for any assignment of variables that makes the body true, the head must also be true for the same assignment

Variables vs constants:
Capitalized tokens like X, Y, Z, U, V are variables.
Lowercase predicate names like parent_of, is_person, aunt_or_uncle_of are predicate symbols.
Entity constants in grounded facts are integers like 0,1,2,3.
An assignment (grounding) means choosing concrete entities for variables, e.g., assigning X=2 and Y=3, and then checking whether the rule body holds under that assignment; if it does, the rule head is entailed under that same assignment.

Built-ins you may see include:
!=(U,V) meaning U and V are not the same entity.

#ARGUMENT 3: entailed_facts (str)
This is a string containing grounded facts that become true after adding cand_fact to facts_program and applying definite_rules_program, excluding facts that were already explicitly present in facts_program (and excluding the candidate fact itself).

Example entailed_facts:

These facts can reflect multi-step inference chains and are the main signal for “non-obvious reasoning unlocked by the candidate.”

#ARGUMENT 4: facts_program (str)
This is the existing story: a string containing grounded facts already present, usually one per line, each ending in a period.

You can view facts_program as the current story graph: each line adds a directed edge (or a self-edge) labeled by the predicate.
For example, is_person(0). can be viewed as a self-edge of type is_person at node 0, and living_in(3,4). is a directed edge of type living_in from node 3 to node 4.
